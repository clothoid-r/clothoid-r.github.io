Who We Are

왜 'Clothoid-R' 인가요?

자동차가 직선 도로에서 급격한 곡선으로 진입할 때 원심력이 급증하여 조향장치가 불안정해집니다. 이를 방지하기 위해 곡률이 완만하게 변하는 전이 곡선을 설계하는데, 이것이 바로 '클로소이드 곡선'입니다.

우리 Clothoid-R은 갑작스러운 자극이나 단절 없이, 복잡하고 예측 불가능한 도심 속에서 자율주행차가 물 흐르듯 부드럽게 주행을 제어할 수 있는 알고리즘 설계와 시스템 통합을 전면 지향합니다.

팀 역할 및 요구 역량 분석

원활한 협업을 위해 인지(Perception), 판단(Planning), 제어(Control)의 3가지 핵심 역할로 팀을 나눕니다. 아래 버튼을 클릭하여 각 파트별 주요 임무와 필요한 기술 스택 비중을 확인하고 본인에게 맞는 역할을 찾아보세요.

인지 파트 주요 임무

  • 3D LiDAR 데이터 처리: Point Cloud 데이터 필터링(Voxel Grid 등) 및 지면 제거 알고리즘 구현
  • 객체 군집화(Clustering): 장애물의 3D 공간상 바운딩 박스 생성 및 위치(X,Y,Z) 추출
  • 카메라 이미지 처리: OpenCV 기반 차선 인식 또는 딥러닝 객체 인지
  • 센서 데이터 융합: LiDAR와 GPS를 융합한 정확한 차량 위치 추정 기초 마련
역량 포커스 다이어그램

🏆 대회 규칙 및 페널티

가장 빠른 랩타임이 우승의 조건이지만, 실수는 치명적인 시간 지연으로 이어집니다. 우측 차트는 기본 랩타임에 페널티가 누적되었을 때의 심각성을 시각적으로 보여줍니다.

  • ⚠️
    장애물 접촉 페널티 꼬깔콘이나 드럼과 충돌할 때마다 기록에 +N초 가산
  • ⚠️
    트랙 이탈 페널티 차량 바퀴가 트랙 밖으로 완전히 벗어나면 이탈 지점 수동 재출발 및 +N초 가산
  • 🚫
    실격 (DNF) 차량 전복, 또는 제어 불능 상태 N분 이상 지속 시
페널티 누적 시뮬레이션 (예시)

안전한 주행이 궁극적으로 가장 빠른 기록을 만듭니다.

System GitHub

3D LiDAR

입력(Input)

Topic: /car1/scan/points

Type: sensor_msgs/msg/PointCloud2

차량 주변의 3D 점군 데이터를 수신하여 장애물의 거리와 형태를 공간적으로 파악합니다.

RGB Camera

입력(Input)

Topic: /car1/camera/image

Type: sensor_msgs/msg/Image

전방 컬러 이미지를 수신하여 차선 인식 및 장애물의 시각적 형태를 파악합니다.

GPS

입력(Input)

Topic: /car1/gps/fix

Type: sensor_msgs/msg/NavSatFix

차량의 현재 글로벌 위치(위도, 경도, 고도) 데이터를 수신하여 전역 매핑에 활용합니다.

Control Cmd

출력(Output)

Topic: /car1/cmd_vel

Type: geometry_msgs/msg/Twist

계산된 최적의 차량 주행 선속도 및 조향 각속도 제어 명령을 시뮬레이터에 전달합니다.
